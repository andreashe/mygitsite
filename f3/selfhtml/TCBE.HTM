Beachten Sie:

Die auf dieser Seite beschriebenen Möglichkeiten sind zwar Bestandteil von HTML 3.2, werden aber leider von keinem der populären heutigen WWW-Browser unterstützt und sind von daher eigentlich wertlos. Sie werden hier eher der Vollständigkeit halber beschrieben.

Standardverweise definieren

Sie können im Kopf einer HTML-Datei Anweisungen notieren, die dem Anwender das Navigieren in großen Informationsbeständen erleichtern sollen. Diese Anweisungen sind dazu gedacht, daß der WWW-Browser, wenn er sie vorfindet, eine entsprechende Button-Leiste oder vergleichbare Bedienelemente anzeigt. Mit dem Anklicken der Buttons kann der Anwender dann bestimmte Sprungverweise zu anderen Dateien ausführen.

Dabei sind folgende Verweisarten vorgesehen:

  • Button mit Sprungverweis zur Home-Page
  • Button mit Sprungverweis zum Inhaltsverzeichnis
  • Button mit Sprungverweis zum Stichwortverzeichnis
  • Button mit Sprungverweis zum Glossar
  • Button mit Sprungverweis zu einer Copyright-Angabe
  • Button mit Sprungverweis zur nächsten Datei in einer "Guided tour"
  • Button mit Sprungverweis zur vorherigen Datei in einer "Guided tour"
  • Button mit Sprungverweis zu einem Hilfekontext
  • Button mit Sprungverweis zu einer beliebigen Textmarke
Sie können pro HTML-Datei ein Set solcher Angaben notieren. Sie können alle, aber auch nur einzelne, gerade benötigte dieser Befehle verwenden.

Beispiel:

 
<head>
<link rel=home href="homepage.htm" title="Einstiegsseite">
<link rel=toc href="inhalt.htm" title="Inhaltsverzeichnis">
<link rel=index href="stichwrt.htm" title="Stichwortverzeichnis">
<link rel=glossary href="glossar.htm" title="Begriffs-Glossar">
<link rel=copyright href="rechte.htm" title="Copyright">
<link rel=next href="augsburg.htm" title="nächste Seite">
<link rel=previous href="aachen.htm" title="vorherige Seite">
<link rel=help href="hilfe.htm" title="Orientierungshilfe">
<link rel=bookmark href="hinweis.htm" title="Neuorientierung">
... andere Angaben im Dateikopf ...
</head>

Erläuterung:

Mit <link rel= leiten Sie einen Standardverweis ein. Dahinter folgt der Typ des Verweises (link = Verweis, rel = relationship = Verwandtschaft). Erlaubte Typen sind:

rel=home steht für: Sprungverweis zur Home-Page
rel=toc steht für: Sprungverweis zum Inhaltsverzeichnis (toc = table of contents = Inhaltsverzeichnis)
rel=index steht für: Sprungverweis zum Stichwortverzeichnis
rel=glossary steht für: Sprungverweis zum Glossar
rel=copyright steht für: Sprungverweis zur Copyright-Angabe
rel=next steht für: Sprungverweis zur nächsten Datei in der "Guided tour" (next = nächste Seite)
rel=previous steht für: Sprungverweis zur vorherigen Datei in der "Guided tour" (previous = vorherige Seite)
rel=help steht für: Sprungverweis zum Hilfekontext (help = Hilfe)
rel=bookmark steht für: Sprungverweis zu einem allgemeinen Orientierungspunkt (bookmark = Lesezeichen)

Hinter dem Verweistyp folgt die Angabe href= zur Bestimmung, wohin der jeweilige Verweis führen soll. Die Angabe sollte in Anführungszeichen stehen. Bei der Angabe gelten die gleichen Regeln wie bei normalen  Verweisen.

Mit der Angabe title= können Sie die Beschriftung der Verweis-Buttons bestimmen. Die Angabe sollte in Anführungszeichen stehen und das Verweisziel nennen.

Mit einem abschließenden > wird der Befehl beendet.

Beachten Sie:

Machen Sie sich nicht die Mühe, mit diesen eigentlich sehr praktischen Befehlen Ihre Projektdateien zu "verlinken". Da kein namhafter WWW-Browser diese Befehle unterstützt, bleiben sie wirkungslos.

Standard-Rückverweise definieren

Ebenso wie Standardverweise sollen Standard-Rückverweise als Buttons oder in vergleichbarer Form dargestellt werden. Bei Standard-Rückverweisen handelt es sich in der Regel um Verweise auf die E-Mail-Adresse des Dokumentautors und auf die Internet-Adresse seiner privaten Homepage.

Beispiel:

<head>
<link rev=made href="mailto:s.muenz@euromail.com" title="Autor">
<link rev=relation href="http://www.autorshome.de/" title="Autoren-Homepage">
... andere Angaben im Dateikopf ...
</head>

Erläuterung:

Mit <link rev= leiten Sie einen Standard-Rückverweis ein. Dahinter folgt der Typ des Verweises (link = Verweis, rev = reverse = umgedreht). Erlaubte Typen sind:

rev=made steht für: Verweis zum Autor, in der Regel eine E-Mail-Adresse
rev=relation steht für: Sprungverweis zur Autoren-Homepage (rel = relationship = Verwandtschaft)

Hinter dem Verweistyp folgt die Angabe href= zur Bestimmung, wohin der jeweilige Verweis führen soll. Die Angabe sollte in Anführungszeichen stehen. Bei der Angabe gelten die gleichen Regeln wie bei normalen  Verweisen bzw. speziell bei  E-Mail-Verweisen.

Mit der Angabe title= können Sie die Beschriftung der Verweis-Buttons bestimmen. Die Angabe sollte in Anführungszeichen stehen und das Verweisziel nennen.

Mit einem abschließenden > wird der Befehl beendet.

Beachten Sie:

Auch hierbei gilt: vergessen Sie diese Angaben, solange kein bekannter WWW-Browser diese Angaben interpretiert.
